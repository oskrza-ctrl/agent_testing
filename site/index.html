El problema

La información valiosa se pierde todos los días

Ideas, pendientes, notas de reuniones y recordatorios se capturan en el momento, pero quedan dispersos sin estructura ni seguimiento.

🎙️

Audios sin procesar

Las notas de voz quedan archivadas sin transcribirse ni analizarse.

📋

Tareas que se pierden

Las acciones derivadas de llamadas o notas nunca llegan a una lista accionable.

💡

Ideas enterradas

Buenas ideas quedan atrapadas en transcripts sin conectarse a proyectos.

📅

Reuniones sin minuta

Conversaciones importantes generan información útil que nadie resume ni guarda.

🗂️

Sin memoria central

No existe un lugar único que relacione notas, tareas, ideas y proyectos.

⚡

Caos documental

Crear un archivo por cada idea genera desorden en lugar de claridad.

Pipeline técnico actual

01
Detección de archivo

El sistema escanea input/ y detecta el tipo de archivo (MP3, TXT, MD).

02
Transcripción (si aplica)

Los archivos MP3 se transcriben con whisper-1. TXT y MD se leen directamente.

03
Análisis de contenido

gpt-4o-mini clasifica la categoría principal y extrae elementos secundarios.

04
Actualización de Knowledge Base

Ideas, tareas, reuniones y recordatorios se guardan en sus archivos correspondientes.

05
Integraciones (V1.3)

Google Tasks y Google Calendar cuando la acción tiene fecha y hora claras.

Reglas funcionales

Decisiones clave del sistema

DEC-007

Calendario vs. Tareas

Fecha + hora clara → Google Calendar
Solo fecha, sin hora → Google Tasks
Ambiguo → Requiere revisión
DEC-008

Ideas en archivo acumulativo

Las ideas no generan un archivo por cada idea. Se agregan al archivo ideas.md para evitar caos documental.

DEC-006

Servicios intercambiables

El código no depende de OpenAI permanentemente. Cada servicio tiene una interfaz que permite cambiar de proveedor sin reescribir el pipeline.

DEC-005

Markdown como base de datos

Sin base de datos relacional. Todo en Markdown plano: legible por humanos, compatible con Obsidian y Google Drive.

DEC-004

Local primero

Se valida el pipeline completo localmente antes de agregar cloud, Google Drive o LangGraph. Menos variables de fallo.

DEC-011

No crear proyectos automáticamente

El sistema puede sugerir proyectos, pero no los crea sin control. Evita generar proyectos basura que contaminen la base de conocimiento.
