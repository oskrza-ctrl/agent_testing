Documentación técnica del estado actual del proyecto: qué está construido, qué hace cada archivo, cómo se relacionan los componentes y qué queda pendiente.

1. Resumen técnico del proyecto

Second Brain Agent es un pipeline local en Python que procesa archivos de audio MP3, los transcribe usando la API de OpenAI y genera un documento Markdown estructurado con ideas, tareas, recordatorios y resumen del contenido.

El sistema está organizado en tres capas:

  • Servicios — lógica de integración con APIs externas (OpenAI Whisper, GPT-4o-mini)
  • Agentes — coordinan y ejecutan pasos discretos del pipeline
  • main.py — punto de entrada que instancia los componentes y lanza el orquestador

2. Arquitectura general

El sistema sigue una arquitectura en capas con separación de responsabilidades. main.py no contiene lógica de negocio: solo instancia servicios y lanza el agente orquestador. Los agentes orquestan el flujo. Los servicios encapsulan las integraciones externas.

Los servicios siguen el patrón Clase Base Abstracta (ABC): cada proveedor (ej. OpenAI) implementa una interfaz genérica. Esto permite cambiar de proveedor sin modificar los agentes ni main.py.

5. Agentes y responsabilidades

AgenteEstadoResponsabilidadMétodo
OrchestratorAgent Implementado Coordina el flujo completo. Maneja errores y decide si archivar. run()
TranscriptionAgent Implementado Delega la transcripción al servicio configurado. run(mp3_path)
AnalysisAgent Implementado Delega el análisis del transcript al servicio configurado. run(transcript)
MarkdownAgent Implementado Garantiza que el Markdown incluya el transcript y lo guarda en disco. run(output_dir, stem, transcript, markdown)
ArchiveAgent Implementado Mueve el MP3 procesado a processed/, con deduplicación por timestamp. run(mp3_path)
Todos los agentes actuales son síncronos y secuenciales. En una futura versión con LangGraph, cada agente se puede convertir en un nodo del grafo con manejo de estado y rutas condicionales entre nodos.

El diseño con ABC garantiza que cambiar de proveedor solo requiere: 1) crear una nueva clase que herede la base, y 2) cambiar una línea en main.py. Los agentes y el orquestador no necesitan modificarse.

7. Lógica actual del sistema

Entrada soportada

Actualmente solo se procesan archivos MP3. El sistema toma el primer archivo *.mp3 encontrado en input/.

TXT y MD aún no están soportados como entradas aunque están en el roadmap de V1.

Prompt de análisis

El prompt vive como constante PROMPT_TEMPLATE en services/analysis/openai_analysis.py. Le indica al modelo que genere estas 8 secciones exactas:

  • # Título sugerido
  • ## Resumen
  • ## Ideas detectadas
  • ## Tareas accionables
  • ## Proyectos relacionados
  • ## Recordatorios
  • ## Tags
  • ## Transcript completo

La instrucción explícita es: si no detecta información en una sección, escribir "No detectado". No inventar.

Manejo de errores

El OrchestratorAgent envuelve los pasos de transcripción, análisis y guardado en un bloque try/except. Si cualquier paso falla:

  • Se imprime el error en terminal.
  • El MP3 original permanece en input/ sin modificación.
  • No se llama a ArchiveAgent.

Deduplicación en archive

Si ya existe un archivo con el mismo nombre en processed/, ArchiveAgent agrega un timestamp al nombre: nota_20260514_204852.mp3.

11. Glosario técnico

ABC (Abstract Base Class)
Clase abstracta en Python que define una interfaz. No se puede instanciar directamente; obliga a que las subclases implementen sus métodos abstractos. Se usa aquí para definir los contratos de TranscriptionService y AnalysisService.
OrchestratorAgent
Agente principal que coordina el flujo completo del pipeline. Es el único que conoce a los demás agentes y decide el orden de ejecución y qué hacer en caso de error.
whisper-1
Modelo de transcripción de audio de OpenAI. Convierte audio en texto. Costo aproximado: $0.006 por minuto de audio.
gpt-4o-mini
Modelo de lenguaje de OpenAI utilizado para analizar el transcript y generar el Markdown. Es el modelo más económico de la familia GPT-4o (~$0.15 por millón de tokens de entrada).
PROMPT_TEMPLATE
Constante en services/analysis/openai_analysis.py que define las instrucciones enviadas a GPT-4o-mini. Incluye la estructura de secciones esperada en el Markdown de salida.
python-dotenv
Librería que carga variables de entorno desde un archivo .env local. Evita hardcodear credenciales en el código.
pathlib.Path
Clase nativa de Python para manipular rutas de sistema de archivos de forma portable entre sistemas operativos.
shutil.move()
Función nativa de Python que mueve un archivo de una ubicación a otra, incluso entre volúmenes distintos. Usada por ArchiveAgent para mover MP3 a processed/.
stem
Nombre del archivo sin extensión. Ejemplo: Path("nota_prueba.mp3").stem retorna "nota_prueba". Usado para nombrar los archivos de salida.
Knowledge_Base
Estructura de carpetas planificada para V1 donde se organizará el conocimiento procesado: ideas, tareas, reuniones, recordatorios y proyectos. Aún no implementada.
LangGraph
Framework de orquestación de agentes basado en grafos de estado. Está en el roadmap de V3 para reemplazar el pipeline secuencial actual con un grafo de nodos con rutas condicionales.
